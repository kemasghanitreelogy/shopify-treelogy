Shopify → QuickBooks Online Integration

A serverless Node.js integration deployed on Vercel that automatically syncs Shopify orders to QuickBooks Online as Sales Receipts. OAuth tokens are persisted in MongoDB Atlas with automatic refresh — no manual re-login needed.

View on GitHub Flow Diagrams

Key Features

📋
Automatic Sales Receipts
Every paid Shopify order automatically becomes a QBO Sales Receipt with correct line items, customer, and currency.
🔒
OAuth 2.0 Auto-Refresh
Access tokens (1hr TTL) auto-refresh using the refresh token (100-day TTL). Login only needed once per 100 days of inactivity.
👤
Customer Auto-Sync
Finds existing QBO customers by email or creates new ones with Shopify name and email data.
📦
Product Auto-Creation
Shopify line items not found in QBO are automatically created as Service-type items with auto-detected income accounts.
🏷
Tax Code Auto-Detection
Queries QBO for valid tax codes, preferring zero-rate or exempt codes. Works with GST, VAT, and tax-free companies.
🛡
HMAC-SHA256 Verification
Every webhook is verified against Shopify's HMAC signature to prevent unauthorized requests.
🔗
Concurrent Refresh Protection
Module-level mutex pattern prevents race conditions when multiple webhooks trigger simultaneous token refreshes.
⚡
Serverless-Ready
Handles Vercel cold starts, MongoDB connection pooling, dynamic IPs, and warm-instance caching.

Complete Integration Flow

The integration has two main flows: a one-time OAuth authorization and a per-order webhook processing pipeline.

1. OAuth 2.0 Authorization Flow (One-Time Setup)

Before the integration can create records in QBO, you must authorize the app. This uses the standard OAuth 2.0 Authorization Code Grant flow.

Step-by-Step OAuth Process

  1. Admin navigates to GET /api/auth/login — Server generates Intuit authorization URL with scope: com.intuit.quickbooks.accounting and state: intuit-test (CSRF protection).
  2. Browser redirects to Intuit — URL: https://appcenter.intuit.com/connect/oauth2 with parameters response_type=code, client_id, redirect_uri, scope, state.
  3. User logs in and grants access — Selects which QBO company to authorize.
  4. Intuit redirects back — To /api/auth/callback?code=AUTH_CODE&realmId=COMPANY_ID&state=intuit-test
  5. Server exchanges code for tokens — Calls oauthClient.createToken(req.url) which POSTs to https://oauth.platform.intuit.com/oauth2/v1/tokens/bearer with grant_type=authorization_code.
  6. Tokens saved to MongoDB — access_token, refresh_token, expires_in (3600), x_refresh_token_expires_in (8726400), and critically tokenCreatedAt as a Number (Unix milliseconds).
  7. Done! — User sees success message. Webhooks can now create QBO records automatically.
🚨 Critical: tokenCreatedAt Must Be a Number The intuit-oauth library checks token validity with arithmetic: createdAt + expires_in * 1000 > Date.now(). If createdAt is a JavaScript Date object instead of a Number, the + operator performs string concatenation instead of addition, producing NaN. This makes isAccessTokenValid() always return false, causing constant unnecessary refreshes that eventually exhaust the refresh token.

Detailed Webhook Steps

  1. HMAC Verification — Compute SHA-256 HMAC of raw request body using SHOPIFY_WEBHOOK_SECRET. Compare Base64-encoded hash with x-shopify-hmac-sha256 header. Reject with 401 if mismatch.
  2. QBO Instance Creation — getQboInstance() loads token from MongoDB, reconstructs with createdAt as Number, checks validity via isAccessTokenValid(), auto-refreshes if expired (with mutex + retry), creates authenticated node-quickbooks client.
  3. Metadata Fetch (Parallel) — Promise.all([getDefaultTaxCode(), getIncomeAccountId()]). Results are cached per warm Vercel instance for subsequent requests.
  4. Customer Resolution — getOrCreateCustomer() searches QBO by PrimaryEmailAddr. If not found, creates a new customer with GivenName, FamilyName, DisplayName (format: "First Last (email)"), and PrimaryEmailAddr.
  5. Line Item Processing — For each Shopify line item: parse price (String → Number), calculate amount (Math.round(price * qty * 100) / 100 to avoid floating point), find or auto-create QBO item as Service type.
  6. Sales Receipt Creation — Build payload with Line[] (SalesItemLineDetail), CustomerRef, CurrencyRef. Call createSalesReceipt().
  7. Response — Return 200 on success (Shopify marks webhook as delivered), 500 on error (Shopify will retry).

4. Concurrent Refresh Protection

In a serverless environment, multiple webhook requests can arrive simultaneously. Without protection, each would attempt to refresh the token, but only the first refresh invalidates the old refresh_token. Subsequent attempts would fail with "invalid_grant".

🔗

Mutex Pattern

qboService.js
// Module-level mutex — shared within a warm Vercel instance
let refreshInProgress = null;

// Inside getQboInstance():
if (!oauthClient.isAccessTokenValid()) {
    if (!refreshInProgress) {
        // First caller initiates refresh
        refreshInProgress = refreshAndSave(realmId).finally(() => {
            refreshInProgress = null;
        });
    }
    // ALL concurrent callers await the SAME promise
    await refreshInProgress;
}

Additionally: If a refresh fails, the service reloads the token from MongoDB. If refresh_token has changed (meaning another Vercel instance successfully refreshed), it uses the updated token. This handles cross-instance race conditions.

index.js — Application Entry Point

Express 5.x Entry Point rawBody Capture

Initializes Express, captures raw request body for HMAC verification, connects to MongoDB, and mounts route handlers.

index.js
require('dotenv').config();
const express = require('express');
const cors = require('cors');
const connectDB = require('./config/db');
const authRoutes = require('./routes/auth');
const webhookRoutes = require('./routes/webhook');

const app = express();

connectDB();   // Non-blocking — begins MongoDB connection

app.use(cors());
app.use(express.json({
    verify: (req, res, buf) => {
        req.rawBody = buf;   // ← Captures raw Buffer BEFORE JSON parsing
    }
}));

// Route mounting
app.use('/api/auth', authRoutes);       // OAuth routes
app.use('/api/webhook', webhookRoutes); // Webhook routes

app.get('/', (req, res) => {
    res.send('Shopify to QBO Integration is running.');
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log(`Server running on port ${PORT}`));
module.exports = app;
ℹ️ Why rawBody? Shopify's HMAC verification requires computing SHA-256 over the exact bytes sent by Shopify. After express.json() parses the body, the JSON may differ from the original (key ordering, whitespace). The verify callback captures the raw Buffer before parsing, storing it as req.rawBody for the HMAC function.

refreshAndSave(realmId) — Token Refresh + Persistence

Refresh Logic
async function refreshAndSave(realmId) {
    // POST to https://oauth.platform.intuit.com/oauth2/v1/tokens/bearer
    // with grant_type=refresh_token
    const authResponse = await oauthClient.refresh();
    const newToken = authResponse.getToken();

    // Persist new tokens (refresh_token rotates — old one is now invalid)
    await Token.findOneAndUpdate(
        { realmId },
        {
            access_token:               newToken.access_token,
            refresh_token:              newToken.refresh_token,   // New rotation
            expires_in:                 newToken.expires_in,
            x_refresh_token_expires_in: newToken.x_refresh_token_expires_in,
            tokenCreatedAt:             newToken.createdAt,       // Number (ms)
        },
        { returnDocument: 'after' }   // Mongoose 9.x (replaces deprecated { new: true })
    );
    return newToken;
}

getQboInstance() — Main Factory Function

Factory
const getQboInstance = async () => {
    // 1. Load latest token from MongoDB
    const savedToken = await Token.findOne().sort({ updatedAt: -1 });
    if (!savedToken) throw new Error('No token in database. Visit /api/auth/login.');

    const realmId = savedToken.realmId;

    // 2. Reconstruct token object — createdAt MUST be Number
    const tokenForOAuth = {
        token_type:                 savedToken.token_type,
        access_token:               savedToken.access_token,
        refresh_token:              savedToken.refresh_token,
        expires_in:                 savedToken.expires_in,
        x_refresh_token_expires_in: savedToken.x_refresh_token_expires_in,
        createdAt:                  savedToken.tokenCreatedAt || Date.now(),
    };
    oauthClient.setToken(tokenForOAuth);

    // 3. Auto-refresh if expired (with mutex + retry)
    if (!oauthClient.isAccessTokenValid()) {
        // ... (mutex logic + DB retry shown in Concurrent Refresh section)
    }

    // 4. Create QuickBooks client — EXACT parameter order required
    const currentToken = oauthClient.getToken();
    return new QuickBooks(
        process.env.QBO_CLIENT_ID,       // 1: consumerKey
        process.env.QBO_CLIENT_SECRET,    // 2: consumerSecret
        currentToken.access_token,        // 3: oauth token (Bearer)
        false,                            // 4: tokenSecret (false = OAuth 2.0)
        realmId,                          // 5: realmId (QBO company ID)
        process.env.QBO_ENVIRONMENT === 'sandbox',  // 6: useSandbox
        process.env.NODE_ENV !== 'production',       // 7: debug logging
        '65',                             // 8: minorversion (API version)
        '2.0',                            // 9: oauthversion — MUST be '2.0'
        currentToken.refresh_token        // 10: refreshToken
    );
};
🚨 Critical: Parameter Order in QuickBooks Constructor The node-quickbooks constructor takes 10 positional parameters. Parameter 8 must be '65' (API minor version) and parameter 9 must be '2.0' (OAuth version). Swapping these causes the library to attempt OAuth 1.0 authentication, which fails with "tokenSecret not defined" because OAuth 1.0 requires a token secret that doesn't exist in OAuth 2.0.

routes/auth.js — OAuth Routes

OAuth 2.0 Authorization Code Grant 
routes/auth.js
const express = require('express');
const router = express.Router();
const OAuthClient = require('intuit-oauth');
const { oauthClient } = require('../services/qboService');
const Token = require('../models/Token');

// ─── Step 1: Redirect to Intuit ───
router.get('/login', (req, res) => {
    const authUri = oauthClient.authorizeUri({
        scope: [OAuthClient.scopes.Accounting],  // com.intuit.quickbooks.accounting
        state: 'intuit-test'                      // CSRF protection
    });
    res.redirect(authUri);
});

// ─── Step 2: Handle callback ───
router.get('/callback', async (req, res) => {
    try {
        const authResponse = await oauthClient.createToken(req.url);
        const tokenData = authResponse.getToken();
        const realmId = oauthClient.getToken().realmId;

        await Token.findOneAndUpdate(
            { realmId },
            {
                realmId,
                token_type:                 tokenData.token_type,
                access_token:               tokenData.access_token,
                refresh_token:              tokenData.refresh_token,
                expires_in:                 tokenData.expires_in,
                x_refresh_token_expires_in: tokenData.x_refresh_token_expires_in,
                tokenCreatedAt:             tokenData.createdAt,   // Number (Unix ms)
            },
            { upsert: true, returnDocument: 'after' }
        );

        res.send('Authorization successful! Token saved. You can close this window.');
    } catch (error) {
        console.error(error);
        res.status(500).send('Authorization failed.');
    }
});

routes/webhook.js — Webhook Handler

Shopify Webhook HMAC Verification Auto-Create Resources Error Handling Caching

The core webhook handler — 253 lines. Receives Shopify orders/create events, verifies authenticity, and creates Sales Receipts in QBO with auto-created customers, items, and auto-detected tax codes.

HMAC-SHA256 Verification

Security
const verifyShopifyWebhook = (req) => {
    const hmacHeader = req.headers['x-shopify-hmac-sha256'];
    const secret = process.env.SHOPIFY_WEBHOOK_SECRET;

    if (!req.rawBody) {
        console.error('⚠️ Error: req.rawBody not found.');
        return false;
    }

    const generatedHash = crypto
        .createHmac('sha256', secret)   // HMAC with webhook secret
        .update(req.rawBody)            // Raw request bytes (NOT parsed JSON)
        .digest('base64');              // Base64 encoding

    return generatedHash === hmacHeader;
};

QBO Error Extraction Helper

Error Handling
// node-quickbooks callback pattern: callback(err, body, res)
// On HTTP errors:
//   err  = axios error object (generic "Request failed with status code 400")
//   body = actual QBO response with Fault.Error details
//
// This helper extracts the REAL error message from all possible locations:

const extractQboError = (err, body) => {
    if (body?.Fault?.Error)                    // 1. QBO error in body (callback arg 2)
        return body.Fault.Error.map(e => `${e.Message} - ${e.Detail}`).join('; ');
    if (err?.Fault?.Error)                     // 2. QBO fault in err object
        return err.Fault.Error.map(e => `${e.Message} - ${e.Detail}`).join('; ');
    if (err?.response?.data?.Fault?.Error)     // 3. Nested in axios response
        return err.response.data.Fault.Error.map(e => `${e.Message} - ${e.Detail}`).join('; ');
    if (err?.response?.data)                   // 4. Raw axios response data
        return JSON.stringify(err.response.data);
    return err?.message || String(err);        // 5. Fallback to generic message
};

Tax Code Auto-Detection (Cached Per Warm Instance)

Tax Codes
let _cachedTaxCodeId = null;

const getDefaultTaxCode = (qbo) => {
    if (process.env.QBO_TAX_CODE) return Promise.resolve(process.env.QBO_TAX_CODE);
    if (_cachedTaxCodeId) return Promise.resolve(_cachedTaxCodeId);

    return new Promise((resolve) => {
        qbo.findTaxCodes([], (err, body) => {
            const codes = body?.QueryResponse?.TaxCode;
            if (codes && codes.length > 0) {
                // Prefer zero-rate/exempt tax codes for flexibility
                const zeroRate = codes.find(c =>
                    c.Active !== false && (
                        /free|exempt|zero|nil|none|nol|bebas/i.test(c.Name) ||
                        c.Name === 'FRE' || c.Name === 'Z' || c.Name === 'NON'
                    )
                );
                _cachedTaxCodeId = zeroRate ? String(zeroRate.Id) : String(codes[0].Id);
            }
            resolve(_cachedTaxCodeId);
        });
    });
};

Income Account Auto-Detection (Cached)

Accounts
let _cachedIncomeAccountId = null;

const getIncomeAccountId = (qbo) => {
    if (process.env.QBO_INCOME_ACCOUNT_ID) return Promise.resolve(process.env.QBO_INCOME_ACCOUNT_ID);
    if (_cachedIncomeAccountId) return Promise.resolve(_cachedIncomeAccountId);

    return new Promise((resolve) => {
        qbo.findAccounts([
            { field: 'AccountType', value: 'Income', operator: '=' }

            const accounts = body?.QueryResponse?.Account;
            if (accounts && accounts.length > 0) {
                const salesAccount = accounts.find(a =>
                    /sales|revenue|pendapatan|penjualan/i.test(a.Name)

                _cachedIncomeAccountId = String(salesAccount.Id);
            }
            resolve(_cachedIncomeAccountId);
        });
    });
};

const getOrCreateCustomer = (qbo, customerData) => {
    return new Promise((resolve, reject) => {
        if (!customerData?.email) return resolve("1");  // Fallback customer

        const email = customerData.email;

        // Search by email using array criteria format
        qbo.findCustomers([
            { field: 'PrimaryEmailAddr', value: email, operator: '=' }
        ], (err, body) => {
            if (err) return reject(new Error('findCustomers: ' + extractQboError(err, body)));

            if (body?.QueryResponse?.Customer?.length > 0) {
                return resolve(body.QueryResponse.Customer[0].Id);
            }

            // Create new customer with unique DisplayName
            const uniqueName = `${customerData.first_name || 'Buyer'} ${customerData.last_name || 'Shopify'} (${email})`;
            qbo.createCustomer({
                GivenName:        customerData.first_name || 'Buyer',
                FamilyName:       customerData.last_name || 'Shopify',
                DisplayName:      uniqueName,
                PrimaryEmailAddr: { Address: email }
            }, (errCreate, bodyCreate) => {
                if (errCreate) return reject(new Error('createCustomer: ' + extractQboError(errCreate, bodyCreate)));
                resolve(bodyCreate.Id);
            });
        });
    });
};

                resolve(bodyCreate.Id);
            });
        });

router.post('/shopify', async (req, res) => {
    if (!verifyShopifyWebhook(req)) return res.status(401).send('Unauthorized');

    try {
        const shopifyOrder = req.body;
        console.log('✅ HMAC Valid. Processing order ID:', shopifyOrder.id);

        const qbo = await getQboInstance();

        // Parallel metadata fetch (cached after first call)
        const [taxCodeId, incomeAccountId] = await Promise.all([
            getDefaultTaxCode(qbo),
            getIncomeAccountId(qbo),
        ]);

        // Customer resolution
        const customerId = await getOrCreateCustomer(qbo, shopifyOrder.customer);

        // Build line items
        const lineItems = [];
        for (const item of shopifyOrder.line_items) {
            const price = parseFloat(item.price) || 0;
            const qty   = parseInt(item.quantity) || 1;
            const amount = Math.round(price * qty * 100) / 100;

            const itemId = await getOrCreateItem(qbo, item.name, price, incomeAccountId);

            const lineDetail = { Qty: qty, UnitPrice: price };
            if (itemId)    lineDetail.ItemRef    = { value: itemId };
            if (taxCodeId) lineDetail.TaxCodeRef = { value: taxCodeId };

            lineItems.push({
                Description:        item.name,
                Amount:             amount,
                DetailType:         "SalesItemLineDetail",
                SalesItemLineDetail: lineDetail,
            });
        }

        // Create Sales Receipt
        const salesReceiptData = {
            Line:        lineItems,
            CustomerRef: { value: customerId },
            CurrencyRef: { value: shopifyOrder.currency },
        };

        const receipt = await new Promise((resolve, reject) => {
            qbo.createSalesReceipt(salesReceiptData, (err, body) => {
                if (err) return reject(new Error('createSalesReceipt: ' + extractQboError(err, body)));
                resolve(body);
            });
        });

        console.log('🚀 Sales Receipt Created! ID:', receipt.Id);
        res.status(200).send('Success');
    } catch (error) {
        console.error('❌ Error:', error.message);
        res.status(500).send('Error processing webhook');
    }
});

node-quickbooks Query Format

⚠️ Important: Array Criteria, Not SQL String node-quickbooks uses an array of criteria objects, NOT raw SQL strings. Passing { Query: "SELECT..." } causes the library to treat "Query" as a column name, resulting in the error "Property Query not found for Entity Customer". 
Correct Format
// ✅ CORRECT — array of criteria objects
qbo.findCustomers([
    { field: 'PrimaryEmailAddr', value: 'user@example.com', operator: '=' }
], callback);

// ❌ WRONG — causes "Property Query not found" error
qbo.findCustomers({ Query: "SELECT * FROM Customer WHERE ..." }, callback);

  • Dynamic IPs: Vercel functions have different IPs per execution — MongoDB Atlas must allow 0.0.0.0/0

Common Errors & Solutions

❌ "Refresh token invalid. Please login again."
Cause: tokenCreatedAt stored as Date instead of Number. intuit-oauth does Date + Number = String (concatenation), so isAccessTokenValid() always returns false. Every request triggers a refresh, eventually exhausting the refresh token.
Fix: Token.js schema must have tokenCreatedAt: { type: Number }. Auth callback must save tokenData.createdAt (already a Number from intuit-oauth).
❌ "tokenSecret not defined"
Cause: QuickBooks constructor parameters 8 and 9 swapped. Without '2.0' as param 9, library defaults to OAuth 1.0 which requires a tokenSecret.
Fix: Parameter 8 = '65' (minorversion), Parameter 9 = '2.0' (oauthversion). Exact order matters.
❌ "Property Query not found for Entity Customer"
Cause: Using findCustomers({ Query: "SELECT..." }). node-quickbooks treats object keys as WHERE column names, so "Query" is sent as a column name.
Fix: Use array criteria: [{ field: 'PrimaryEmailAddr', value: email, operator: '=' }]
❌ "Make sure all your transactions have a GST rate"
Cause: Hardcoded TaxCodeRef: "NON" is invalid for GST-enabled QBO companies (e.g., Australia, India).
Fix: Use getDefaultTaxCode() which queries QBO for valid tax codes and prefers zero-rate/exempt codes. Or set QBO_TAX_CODE env var.
❌ MongoDB Connection Timeout
Cause: Vercel dynamic IPs not whitelisted in Atlas, or cold start exceeds default timeout, or Atlas cluster is paused (free tier pauses after 60 days of inactivity).
Fix: Atlas Network Access → 0.0.0.0/0. Set serverSelectionTimeoutMS: 10000. Resume paused cluster.
❌ "Request failed with status code 400" (generic message)
Cause: Default error handling only reads err.message (generic Axios error). The actual QBO error details are in the second callback argument (body.Fault.Error).
Fix: Use extractQboError(err, body) helper which checks body.Fault.Error first, then err.Fault.Error, then err.response.data.Fault.Error.
❌ Mongoose Deprecation Warning: { new: true }
Cause: Mongoose 9.x deprecated { new: true } option in findOneAndUpdate().
Fix: Replace with { returnDocument: 'after' } in all findOneAndUpdate() calls.
